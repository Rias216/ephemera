1. CURRENT STATE ASSESSMENT
================================================================================

WHAT EPHEMERA HAS TODAY:
  - Observe-Plan-Act-Verify agent loop (internal/agent/agent.go)
  - 11 built-in tools: list_files, tree, read_file, search, git_status,
    git_diff, apply_patch, replace_in_file, shell, go_test, delegate
  - 5 LLM providers: OpenAI, Anthropic, Ollama, Codex, OpenAI-compatible
  - SQLite session persistence
  - Delegate sub-agent for independent review
  - Doom-loop detection and approval gating
  - Project memory via .ephemera/ files
  - Built-in deterministic eval (7 test cases)
  - Approval policies: chat, read-only, approve-writes, workspace-write, auto

WHAT EPHEMERA LACKS vs. FRONTIER:
  - No MCP (Model Context Protocol) support
  - No web browsing / URL fetching tools
  - No vector-based semantic memory
  - No containerized/sandboxed execution
  - No parallel tool dispatch
  - No plugin/extension architecture
  - No tree-of-thought or advanced planning
  - No self-reflection / learning from errors
  - No multi-modal support (images, PDFs)
  - No streaming tool results
  - No tool caching / memoization
  - No distributed agent coordination
  - No observability / metrics layer
  - No graceful degradation under token pressure
  - No structured output validation (JSON schema enforcement)

10. RISK MITIGATION
================================================================================

RISK: Increased complexity makes debugging harder
  MITIGATION: Comprehensive observability from Phase 6, structured logging,
              distributed tracing, and detailed error messages.

RISK: Container sandbox adds Docker dependency
  MITIGATION: Make sandboxing optional, fall back to current shell execution
              when Docker is unavailable. Provide clear setup instructions.

RISK: Parallel tool execution introduces race conditions
  MITIGATION: Extensive concurrency testing, proper mutex usage, immutable
              shared state where possible, and thorough code review.

RISK: MCP integration increases startup time
  MITIGATION: Lazy MCP server connection, tool schema caching, and async
              discovery. Don't block startup on slow MCP servers.

RISK: Vector memory increases storage and cost
  MITIGATION: Local embedding option via Ollama, configurable memory limits,
              and cost tracking for cloud embeddings.

RISK: Plugin system creates security surface
  MITIGATION: Sandboxed plugin execution, capability-based permissions,
              and mandatory code signing for marketplace plugins.

RISK: Multi-agent coordination overhead
  MITIGATION: Start with simple delegation, add complexity gradually, and
              always allow single-agent fallback mode.

RISK: Context window optimization loses important information
  MITIGATION: User-configurable retention policies, always preserve recent
              turns and file states, and provide context inspection tools.
